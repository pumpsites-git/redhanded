1. High-Level System Architecture

Three-layer architecture: external data sources feed an ingestion pipeline, which normalizes and stores data. The scoring engine calculates accountability scores. The frontend serves search, profiles, and community features.

2. Data Ingestion Pipeline — Step by Step

Each data source follows a specific path. The pipeline runs nightly to keep scores fresh. Federal data is clean and API-accessible; state data requires per-state scrapers (Phase 2).

3. Accountability Score — How It's Calculated

The Accountability Score (0-100) is a weighted composite of four factors. Each factor is normalized to 0-100, then weighted. The score intentionally surfaces judges whose decisions lead to worse public safety outcomes.

4. User Experience Flow

Three main user journeys: search/discover, deep-dive on a judge, and community participation. The election alert system drives civic engagement.
